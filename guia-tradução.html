Estrutura de arquivos

As traduções são instaladas por meios de pastas. Elas ficam dentro do diretótio /translations, ao lado do jogo (TalesOfAndrogyny.exe).
Por exemplo, para a tradução em pt-br, adicionamos a pasta pt-br dentro do diretótio /translations e temos a seguinte árvore de diretótios 

📜Tales of Androgyny.exe
📂/translations
 ┗ 📂/pt-br
    ┣ 📜manifest.json
    ┗ 📂/assets
       ┣ 📂/script
       ┃  ┗ 📜encounters.json
       ┗ 📂/translation
          ┗ 📜strings.properties

manifest.json

O arquivo de manifest.json contém os seguintes dados:

{
    "languageName": "Português",
    "localeId": "pt-BR"
}

Onde:

Propiedade Descrição
languageName Nome do idioma que irá aparecer nas configurações
localeId locale ID para a língua (veja a lista de id's: https://www.localeplanet.com/icu/). Serve para localizar a tradução, sendo usado para os plurais e as demais formatações do idioma.

Assets

Alguns assests (a maioria sendo imagens) no jogo têm textos e vão precisar ser alterados para que sejam traduzidos. Para traduzi-los, encontre o asset que deseja na pasta /assets dentro do arquivo TalesOfAndrogyny.jar (talvez você precise extraí-lo antes), então, edite o mesmo trocando o texto para a tradução, e em seguida mova-o para a pasta /asset dentro da pasta da tradução com o mesmo nome e subdiretótiio do original.
Por exemplo: Após editar o asset Static_Elements.png localizado na pasta assets/ui/option/Static_Elements.png, ele deverá ser movido para o diretótio translations/es/assets/ui/option/Static_Elements.png.

Para ter acesso aos assets do jogo, você precisa abrir o arquivo TalesOfAndrogyny.jar com algum programa de descompactação, como 7-Zip ou WinRAR. Também é possível renomear o arquivo para TalesOfAndrogyny.zip e extrair em seguida.
Caso você use linux, você pode apenas clicar com o botão direito do mouse no arquivo e clicar em "extrair aqui" (o nome pode mudar dependendo da distro)

Textos

Há dois tipos principais de texto no jogo, sendo eles:

  • Encounters.json - arquivo onde há todos os encontros no jogo. Ou seja, as partes da história que são mostradas no estilo "visual novel", como fala dos personagens e interações, incluindo partes da interface e intruções.
    Exemplo: 
    "text": "Depois de uma curta caminhada, não encontrando ninguém pelo caminho (felizmente), você vê a casa de campo à distância. A fumaça sai lentamente da chaminé - um fogo fraco aceso lá dentro. Ao se aproximar da porta da frente, você ouve uma voz fraca e abafada; um homem está falando."}
  • strings.properties - arquivo onde está outros textos do jogo, como menus e informações de batalha. 
    checktype.tight_butthole.success=Seu cu é apertado!

Nota: todos os textos devem ser salvos usando UTF-8. Certifique-se se seu editor está configurado para salvar aquivos usando essa codificação. Caso contrário os caracteres que não fazem parte do alfabeto latino irão causar problemas. Use apenas UTF-8.

encounter.json

O arquivo encounter.json encontrado no diretório assets/script/encounters.json é responsável por armazenar todos os encontros definindos por nome, sendo cada encontro uma lista de ítens.
Por exemplo: 

"REUNIÃO DE GOBLIN": [
    {"text": "Em guarda, você enfrenta o familiar sinal de alerta."},
    {"animatedForeground": "GOBLIN",
    "text": "Você encontra o mesmo goblin de antes! Ela sorri para você, mostrando seus dentes afiados, e ri alto enlouqecidamente enquanto ataca!"}
],

Cada item pode ter as seguintes propriedades que devem ser traduzidas:

  • speaker - Indica quem está falando. O nome aparece na bolha na parte superior esquerda na caixa.
    Nota: a propiedade speaker não pode ser alterada caso o seu valor seja Hiro, pois o mesmo é alterado pelo nome do personagem durante a execução do jogo.
  • text - Indica o texto que será mostrado na caixa de diálogo principal.
  • chatter - Indica o texto que aparece na caixa de mensagem lateral às vezes usado por, por exemplo, membros do grupo.

Todas as outras propiedades são ignoradas e podem ser descartadas.

Cada encontro individual na sua tradução vai ser mesclado com o texto em inglês base com o mesmo ID. Ou seja, se a sua tradução não tiver o encontro, o jogo automáticamente irá usar o encontro em inglês correspondente. Além disso, como as mudanças nos encontros podem quebrar o fluxo das cenas, se sua tradução não tiver a mesma quantodade de itens na lista (por exemplo, porque o jogo foi alterado para ter mais conteúdo que o jogo base), a sua versão do encontro será ignorada e deverá ser atualizada.

O arquivo de strings

O arquivo de strings tem o formato .properties (https://en.wikipedia.org/wiki/.properties) pode ser encontrado no diretório assets/translation/strings.properties. Esse arquivo guarda quase todos os textos usados nos menus, notificações etc. Geralmente usado por qualquer texto que não faz parte do texto principal de um encontro.

O arquivo usa um formato key=value, ou seja, quando o jogo precisar mostrar a string, ele vai entrar no arquivo e procurar a key correspondente, e logo em seguida mostrando seu valor.
Por exemplo: Se o jogo precisar mostrar o menu "carregar", ele vai procurar pela key mainmenu.load e sua definição mainmenu.load=Carregar e mostrar o valor Carregar

Se não tiver um arquivo de strings no pacote de tradução, ou não houver uma key específica, o jogo usará o valor padrão atribuido no jogo base.

Algumas strings não são textos fixos, podendo ter palavras e expressões diferentes para certos casos. Para isso é utilizado um sistema de preenchimento. Leia o próximo tópico para saber como usar.

Preenchimento

Algumas strings precisam ter algo inserido dinamicamente dependendo da situação que o player se encontra.
Por exemplo:

  • item.potion.description.mana=Restaura {0} de mana. - A quantidade de mana precisa ser inserida.
  • skillselectionscene.you_have_removed_someperk=Você removeu {0}. - o nome da vantagem precisa ser inserido

Nesses casos, o jogo olha para a string através da key, mas troca o número (ou outra string) por uma string correspondente a situação.
Em alguns casos pode ser necessário vários preenchimentos.

  • skillselectionscene.you_have_reduce_someperk_to_rank_x=Você reduziu {0} para o rank {1}. - põe o nome de uma vantagem no 0 e um número no 1.

O jogo especifica os preenchimentos em uma determinada ordem (começando em 0 e não em 1). Assim o {0} é recebe o valor que foi determinado, o mesmo para {1}, {2} e em diante.

Alguns idiomas precisam que os preenchimentos sejam colocados em locais diferentes, depenendo da sentença. Enquanto você estiver traduzindo uma string, determine um local adequado para o preenchimento e use a sintáxe para inseri-lo.

No geral, deve ser fácil de entender o que os preenchimentos fazem seguindo o contexto.
Algumas keys, entretanto, possuem nomes como shopscene.you_purchase_the_x, que pode implicar no entendimento do que o preenchimento faz.
Para as outras, a sentenca em inglês já deve ser o suficiente para contextualizar e facilitar o entendimento.

Complicações com preenchimentos

Dependendo do idioma, alguns preenchimentos não são simples de inserir o número ou a palavra na sentença. Muitos idiomas têm plural, gêneros etc.

Plurais

Enquanto a sentença precisar ser muito baseada em plurais, use a sintaxe de plurais.
Aqui está um exemplo: 

mutation.cured_x_bleed_points={0} curada com {0, plural, one{ponto.} other{pontos.}} de sangramento

O número de pontos de sangramento é 0, que será preenchido de acordo com a sintáxe de plurais, o qual decidirá se deverá ser usado uma palavra no singular ou no plural. Para entender melhor a sintáxe, veja abaixo (comece da esquerda): 

{0, plural, one{ponto.} other{pontos.}}
 ▲    ▲      ▲    ▲       ▲      ▲
 │    │      │    │       │      │
 │    │      │    │       │      │
 │    │      │    │       │      │
 │    │      │    │       │      │ Use essa palavra (plural).
 │    │      │    │       │
 │    │      │    │       │Se o preenchimento for qualquer outro número...
 │    │      │    │
 │    │      │    │ Use essa palavra (singular).
 │    │      │
 │    │      │ se o preenchimento for 1...
 │    │
 │    │ a sintáxe do plural.
 │
 │ O número de preenchimento.

O one e other fazem parte das regras de plural. As regras podem variar dependendo do idioma. Para inglês e português por exemplo, é possível usar one para o singular e other para todos os outros casos, incluindo o plural.

Para usar as regras do seu idioma, especifique o localeId no arquivo manifest.json e consulte o idioma na tabela de regra de plural do ICU (leve em consideração somente o tipo cardinal, ignore o ordinal e range )

Gênero e pronomes

Para algumas strings é preciso saber o gênero do personagem. Normalmente, nesses casos um dos preenchimentos é um tipo especial chamado de conjunto de pronome. Quando há um pronome definido como um dos preenchimentos, pode ser usado um dos seguintes tokens para inserir pronomes.

  • {nom0} - substituído por pronouns.<gender>.nominative, por exemplo: ele, ela
  • {poss0} - substituído por pronouns.<gender>.possessive, por exemplo: dele, dela
  • {obj0} - substituído por pronouns.<gender>.objective, por exemplo: ele, ela

o 0 acima é apenas um exemplo. Se o conjunto de pronomes estiver em um índice diferente, basta usar esse índice - {nom1}, {poss2}, etc

Se você precisar que as palavras sejam maiúsculas, existem versões alternativas que colocam a primeira letra do pronome em maiúscula: {Nom0}, {Poss0}, {Obj0}.

Se isso não for suficiente para oferecer suporte a pronomes e gênero para seu idioma, você poderá usar a seleção para criar uma condicional em um preenchimento, semelhante à plural acima. Aqui estão dois exemplos:

  • {0, select, m{He leaves the room.} other{She leaves the room.}} - picks "He leaves the room." if 0 is "m", otherwise picks "She leaves the room."
  • {1, select, f{She walks up to the counter.} m{She walks up to the counter.}} - picks "She walks up to the counter." if 1 is "f", and picks "She walks up to the counter." if 1 is "m"

This works because a pronoun set fill is automatically turned into m or f.

See the diagram above explaining the plural syntax if this looks confusing. This works much the same, but the difference is it checks text instead of numbers.

Fill complications - (advanced!) Solving grammatical gender, case, and additional requirements with Extra Keys

For advanced requirements for certain languages, we have a system of extra keys. Extra keys allow you to define additional variants of strings, or additional attributes that you can then do conditional logic on.

Extra keys can also be used in conditionals. For example, if you have a sentence that depends on gender for something that isn't already configured to support gender (characters), such as grammatical gender in Spanish: 

library=biblioteca
book=libro
where_is_the_x=¿Donde está el {0}?

You would run into an issue as the sentence for where_is_the_x has to be different depending on whether it's about a book or a library. To solve this, you can add extra information in an extra key, and then use that extra key in a select conditional much like we did for character gender: 

library=biblioteca
library.gender=f
book=libro
book.gender=m
where_is_the_x=¿Donde está {0.gender, select, m{el {0}} f{la {0}}}?

Note: the names of extra keys should start with a letter or underscore and should only contain letters from the base latin alphabet, numbers, and underscores.

Fonts

The fonts included with the game may not contain all the characters required for your language. If that is the case, you will need to find fonts that do and replace two types of font files (using asset replacement):

  • .ttf - A TrueType font, similar to the ones you would normally download to install on your computer.
  • .fnt + .png - A generated bitmap font. You will need to use the Hiero tool to create these.
